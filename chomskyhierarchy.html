§ 01 — Introduction

What is the Chomsky Hierarchy?

In 1956, linguist and mathematician Noam Chomsky introduced a containment hierarchy of classes of formal grammars — rule-based systems for generating strings over an alphabet. Each class is strictly more expressive than the one inside it, and each corresponds to a class of abstract computational machines.

The hierarchy has four levels, numbered Type 0 through Type 3 from most to least powerful. A language in a lower type is also in all higher types — the classes are nested, not disjoint.

Each class is defined by the form of production rules allowed in its grammar. As we move outward, the rules become less constrained — and the machines required to recognise the resulting languages become more powerful.

TYPE 3
Regular Languages
Recognized by: Finite Automaton (DFA / NFA)
+

Regular languages are the simplest class in the hierarchy. They are described by regular expressions and recognized by finite automata — machines with no auxiliary memory. The machine reads the input left-to-right, transitions between a finite set of states, and accepts or rejects based solely on the final state.

The key restriction is that finite automata cannot count or match nested structures — they have no stack or tape. This means any language requiring unbounded memory is outside this class.

Grammar Production Rules
// Right-linear grammar (one of two equivalent forms)
A → a // single terminal
A → aB // terminal followed by non-terminal
A → ε // empty string

// Left side: exactly one non-terminal
// Right side: at most one non-terminal, always rightmost
Example Grammar — aⁿ (any number of a's)
S → aS | ε
// Generates: ε, a, aa, aaa, aaaa, ...
Language Examples
aⁿ — strings of a's (a|b)*abb — ends in abb valid email addresses C identifiers binary multiples of 3 decimal integers IP addresses even-length strings
Closure Properties

Regular languages are closed under union, intersection, complement, concatenation, Kleene star, and reversal. These operations produce another regular language.

Fundamental limitation: Cannot recognize aⁿbⁿ (equal numbers of a's and b's). This requires counting — which needs unbounded memory. Proved formally by the Pumping Lemma for Regular Languages: for any sufficiently long string, a portion can be "pumped" and the language must still contain it.
TYPE 2
Context-Free Languages
Recognized by: Pushdown Automaton (PDA)
+

Context-free languages (CFLs) are recognized by pushdown automata — finite automata augmented with an infinite stack. The stack enables the machine to count and handle nested structures of arbitrary depth, making this class ideal for programming language syntax and arithmetic expressions.

Context-free grammars (CFGs) are the theoretical foundation for parser generators like ANTLR, YACC, and Bison used in compiler construction.

Grammar Production Rules
A → γ
// Left side: exactly one non-terminal A
// Right side: any string γ of terminals and non-terminals

// No restriction on what surrounds A — hence "context-free"
Example Grammar — aⁿbⁿ (equal a's and b's)
S → aSb | ε
// Generates: ε, ab, aabb, aaabbb, ...
Example Grammar — Balanced Parentheses
S → SS | (S) | ε
// Generates: (), (()), ()(), ((())), ...
Language Examples
aⁿbⁿ balanced parentheses arithmetic expressions most programming language syntax palindromes over {a,b} simple natural language syntax XML / JSON structure
Chomsky Normal Form (CNF)

Every CFL has a grammar in CNF where every rule takes one of exactly two forms:

A → BC two non-terminals
A → a single terminal

CNF is useful for parsing algorithms like CYK (Cocke–Younger–Kasami) which runs in O(n³) time.

Fundamental limitation: Cannot recognize aⁿbⁿcⁿ. The PDA can push n copies of 'a' onto the stack and pop while reading 'b', but has no memory left to count 'c'. Cross-serial dependencies (as in Swiss German) are also beyond this class.
TYPE 1
Context-Sensitive Languages
Recognized by: Linear Bounded Automaton (LBA)
+

Context-sensitive languages (CSLs) are recognized by linear bounded automata — Turing machines whose tape is restricted to the length of the input (times a constant). The key feature of context-sensitive grammars is that production rules can only replace a non-terminal in the context of its surrounding symbols.

This class is computationally decidable (the machine always halts), but recognition is PSPACE-complete — believed to require exponential time in the worst case.

Grammar Production Rules
αAβ → αγβ
// A is a non-terminal in context α _ β
// γ must be non-empty (no erasure — rules are monotonic)
// |αγβ| ≥ |αAβ| — right side no shorter than left side
Example Grammar — aⁿbⁿcⁿ
S → aSBC | aBC
CB → BC
aB → ab
bB → bb
bC → bc
cC → cc
// Generates: abc, aabbcc, aaabbbccc, ...
Language Examples
aⁿbⁿcⁿ {ww | w ∈ Σ*} copy language Swiss German cross-serial deps aⁿ² — square lengths triple-copy language
Natural Language Connection

Chomsky originally proposed this class to model natural language syntax. Modern computational linguistics suggests natural languages may be mildly context-sensitive — between context-free and fully context-sensitive. Tree-adjoining grammars (TAGs) are a prominent formalism in this gap.

Decidability: Unlike Type 0, membership in a context-sensitive language is always decidable — the LBA always halts. However, it is PSPACE-complete, meaning no polynomial-time algorithm is known unless P = PSPACE.
TYPE 0
Recursively Enumerable Languages
Recognized by: Unrestricted Turing Machine
+

Recursively enumerable (RE) languages are the most expressive class — recognized by unrestricted Turing machines. Any computation that can be described algorithmically generates or recognizes an RE language. There are no restrictions on grammar productions.

A critical distinction exists within this class: some RE languages are recursive (decidable) — the Turing machine always halts. Others are only semi-decidable — the machine halts and accepts members, but may loop forever on non-members.

Grammar Production Rules
α → β
// Left side: any non-empty string of terminals and non-terminals
// Right side: any string (including empty)
// No monotonicity constraint — strings can shrink
Sub-classes
Recursive (Decidable)TM always halts. Closed under complement.
RE but not RecursiveTM may loop on rejection. Not closed under complement.
Language Examples
Halting problem (semi-decidable) Post Correspondence Problem Universal language Lᵤ Hilbert's 10th problem all Turing-recognizable languages
Beyond RE

Not all languages are recursively enumerable. The complement of the halting problem is an example of a language that is not RE — no Turing machine can recognize it. Uncountably many languages exist, but only countably many Turing machines.

Church–Turing Thesis: Any effectively computable function can be computed by a Turing machine. Type 0 grammars represent the limit of what can be computed at all. Beyond this boundary lies the undecidable.
§ 03 — Computational Models

Automata Diagrams

Each language class corresponds to a class of abstract computational machines. The diagrams below illustrate their structure and operation.

A Deterministic Finite Automaton (DFA) has a finite set of states, reads input left-to-right one symbol at a time, and transitions between states. It has no memory beyond its current state. The example below recognizes strings over {a,b} that end in "ab".

The DFA has three states. It accepts any string ending in "ab". No stack, no tape — just current state and input symbol determine the next transition.

A Pushdown Automaton (PDA) extends the DFA with an infinite stack. The transition function can read the top of stack, pop it, and push new symbols. This gives it the power to recognize nested structures. The example recognizes aⁿbⁿ.

The PDA pushes symbol 'A' onto the stack for each 'a' read, then pops one 'A' for each 'b'. If the stack is empty precisely when the input is exhausted, the string is accepted — ensuring equal counts.

A Linear Bounded Automaton (LBA) is a Turing machine whose tape is limited to the input length (times a constant). It can read and write, move left and right, but cannot use more space than the input provides.

The read/write head can move left and right within the bounded tape. The LBA recognizes aⁿbⁿcⁿ by marking and counting — something a PDA cannot do with only a stack.

A Turing Machine has an infinite read/write tape, a finite control unit, and a head that can move in both directions. It is the universal model of computation. The tape is infinite — unlike the LBA, there is no bound on space usage.

The Turing machine's transition function δ maps (state, tape symbol) → (new state, write symbol, move direction). The infinite tape and bidirectional head give it universal computational power. Acceptance may require the machine to run forever on some inputs.

§ 05 — Deeper Theory

Pumping Lemmas & Key Theorems

Pumping lemmas are the primary tools for proving that a language does not belong to a given class. They exploit structural properties that all sufficiently long strings in a class must satisfy.

Pumping Lemma — Regular Languages

If L is a regular language, there exists a constant p ≥ 1 (the "pumping length") such that every string w ∈ L with |w| ≥ p can be split into three parts w = xyz satisfying:

1. |y| ≥ 1 (y is non-empty) 2. |xy| ≤ p (xy is short) 3. ∀ i ≥ 0: xyⁱz ∈ L (y can be pumped)

To prove a language is not regular: assume it is, find a string that violates one of these conditions for any split, and derive a contradiction.

Pumping Lemma — Context-Free Languages

If L is a CFL, there exists p ≥ 1 such that every w ∈ L with |w| ≥ p can be split into w = uvxyz satisfying:

1. |vy| ≥ 1 (at least one of v, y is non-empty) 2. |vxy| ≤ p (middle portion is short) 3. ∀ i ≥ 0: uvⁱxyⁱz ∈ L (v and y pumped together)

Used to show aⁿbⁿcⁿ is not context-free: pumping any split produces an imbalance in at least one count.

Applications in Computer Science

Regular languages underpin lexical analysis — the first phase of every compiler. Tools like lex and flex compile regular expressions to DFAs for tokenizing source code.

Context-free grammars describe the syntax of virtually every programming language. Parser generators like YACC, Bison, and ANTLR take a CFG and produce an efficient parser automatically. JSON, HTML, and XML are defined by CFGs (though real parsers add extensions).

Context-sensitive and RE classes appear in theoretical computer science when studying computability and complexity — which problems can be solved at all, and how efficiently.
